## 代码级别的标准

核心：考察其是否具备解决深层、复杂技术问题的能力，以及代码是否具备生产级质量与可维护性。

简而言之，也就是一些细节层面的考察。

### 代码质量与规范

主要关注 MR

1. 能写出合格、干净的代码
2. 能 Review 别人的代码，并给出有效评论（避免无脑 Approve)
3. 能接受别人的 Review 并根据实际情况修改代码或做出解释
4. 对代码质量有追求
5. 能主动优化系统存量代码
6. 写代码会关注性能、技术前瞻、接口设计

### 复杂问题定位与解决

主要关注值班 oncall 处理

1. 能心平气和沟通、认真对待用户、运营、产品的 oncall 并跟踪解决
2. 熟练使用公司平台的排查能力，处理排查线上问题
   - 包括不限于 sentry/logging/profile/grafana 等
3. 能追踪并彻底解决一个疑难问题
4. 能主导某个故障的根因分析，并推动解决
5. 在事故复盘时，能超越表面现象，指出系统设计、流程或协同上的根本原因，并提出长效改进措施。
6. 能够在需求中对解决问题进行代码补充
   - 合理打日志+打指标
   - 灰度与放量
   - 产出一些缺失的规范

### AI 使用经验

主要关注 MR 以及观察日常使用情况

1. 能够熟练使用 Cursor / Claude Code 等类似 AI Coding Agent 产品写代码
2. 能够评估 AI 生成代码的质量，能够避免提交 AI 生成的烂代码
3. 能够分享 1-5 条 Vibe coding 使用心得
4. 尝试使用文档驱动模式开发某个小型、中型需求
5. 能够利用 coding agent 排查问题
6. 能够接触更新的 AI 知识点，并在团队内介绍

## 工程架构师的标准

核心：考察其是否具备系统级思维，能否设计出支撑业务发展、兼顾长期演进的合理架构。

简而言之，这部分不考察具体的代码技术，更关注合理的架构设计、架构意识。

### 架构设计能力

主要关注 RFC

1. 写出 N 篇质量较高的大型/中型需求 RFC
2. 能 Review 其他人的 RFC 方案，并给出有效提问/评论
3. 能根据其他人的 Review 沟通调整方案或者做出有效解释
4. 能合理评估需求复杂度和工程排期
5. 能够根据需求做好技术方案的取舍
6. 技术方案能够有前瞻性、健壮性

### 复杂项目实施

核心考察：能否主导高风险、高复杂度的系统性变更，并确保平稳落地。

主要关注技术债务类技术迁移需求

1. 能够充分评估历史现状、未来期望和改造风险
2. 能够合理评估并限定技术改造范围、周期、依赖
3. 能够合理规划与拆解复杂任务
4. 能够主动推动任务完成
5. 能够主动评估处理历史遗留问题

### 对外合作与沟通

主要关注群聊、需求评审、联调

1. 熟悉公司架构、部门、人，知道谁负责，该找谁，什么问题大概是谁在负责
2. 能和其他团队联调通，并上线完成一个需求
3. 能提前和前端沟通技术方案
4. 能够心平气和与产品、外部同事就事论事沟通
5. 能够对产品需求评审时做出有效提问/评论
6. 能够对具体业务需求方案有自己的见解/思路
7. 能够跨部门推动一些事情，推广自己的技术影响力

## 文化认同与软性素质

这部分我就不列公司对应的价值观列表了，我会都放到一起，并作为一个软性的衡量标准。

### 文化认同

这部分我就不列公司对应的价值观列表了，可以自行阅读 员工手册 的部分。

需要注意，价值观也是自己的精神需求，这些精神层面的东西，需要自己的认可才有用。想明白自己要干什么，自己的人生该何去何从，不能不思考 🤔。

### 基本素质

**信息安全**

1. 不要因为自己的身份(公司职务)知道某事，就对外泄漏相关信息。
2. 不要泄漏公司的产出的代码/文档/截图/未来规划
3. 不要借助平台给自己账号/他人账号做额外的事情
4. 严肃对待信息泄漏，保护用户信息

**心平气和**

1. 能够心平气和与产品、外部同事就事论事沟通
2. 能够认真对待用户的反馈、oncall、江湖救急

**信息同步**

1. 尽量减少私聊，凡事尽量公开群聊（光明正大地工作/回复）
2. 有重要信息的群/突发事件的群，及时拉上相关同事、上级以做沟通
3. 聊天需求/一句话需求，尽量以 task /需求 形式落地追踪，并添加对应的相关同事、上级关注

**向上求助**

1. 不懂/拿不定/有疑问/外部重大请求，避免未评估/未沟通的对外承诺
2. 减少个人单独决策，及时拉群，向上求助

## 做的更好？

### 关注外部

1. 关心每一个同事正在做什么，知道谁在做什么需求
2. 关注公司/部门的 OKR，关注更上级的 OKR (可以搜索【怎样当好一名师长】 参考阅读)

### 个人价值实现

1. 我觉得系统/代码/公司 哪里哪里不好
2. 我觉得哪里应该怎么做，我要怎么做
3. 多想想我来这个公司要做什么？

### 未来规划与长远发展

1. 我们的产品、服务、代码要怎么更好的用 AI 呢？
2. 多想想手上的产品、代码、服务未来要何去何从（半年？一年要怎么做？两年要怎么做？）
